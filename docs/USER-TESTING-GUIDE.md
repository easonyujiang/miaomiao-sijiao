# 妙喵私教 — 真实环境搭建与用户测试手册

本文档面向**真实用户视角**的测试人员，所有操作通过浏览器完成，不涉及命令行测试工具。

---

## 一、环境搭建（一次性操作）

### 1.1 前置条件

| 依赖 | 版本要求 | 检查方式 |
|------|----------|----------|
| Python | >= 3.11 | 终端输入 `python --version` |
| Node.js | >= 18 | 终端输入 `node --version` |
| 任一 LLM API Key | Kimi 或 DeepSeek | 从对应平台获取 |

### 1.2 后端启动

```powershell
# 进入服务端目录
cd E:\Create\Code\ewa\server

# 创建虚拟环境并安装依赖（首次）
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -e ".[dev]"

# 配置环境变量（首次）
Copy-Item .env.example .env
# 用记事本打开 .env，填入至少一个 API Key：
#   MOONSHOT_API_KEY=sk-你的真实key
#   DEEPSEEK_API_KEY=sk-你的真实key（可选）

# 启动后端
python run.py
```

看到以下输出说明启动成功：
```
妙喵私教 starting up...
妙喵私教 ready. DB: E:\Create\Code\ewa\server\data\miaomiao.db
```

后端地址：**http://localhost:8000**（本地开发）或 **http://8.130.190.169:8000**（生产服务器）

> **插件连接地址**：插件默认连接生产服务器 `http://8.130.190.169:8000`，无需改代码。如需切换（例如改回本机后端 `http://localhost:8000`），点击浏览器工具栏的插件图标打开 popup，在底部「服务器地址」设置中修改即可——配置保存在 chrome.storage，刷新页面后生效（注意：目前仅 API 请求地址生效，面板中的网站跳转链接仍用默认值，见 ISSUE-011）。

### 1.3 前端启动（可选，用于开发模式）

```powershell
# 新开一个终端
cd E:\Create\Code\ewa\server\frontend
npm install
npm run dev
```

前端地址：**http://localhost:3000**（自动代理 API 请求到后端）

### 1.4 确认服务正常

打开浏览器访问 http://localhost:8000/health （本地开发）或 http://8.130.190.169:8000/health （生产服务器），应看到：
```json
{"status":"ok","site":{"seeded":true},"timestamp":"2026-07-18T..."}
```

如果 `seeded` 为 `true`，说明种子数据已加载完毕。

> **生产环境网站入口**：
> - 文字浏览与聊天：**http://8.130.190.169:8000**
> - 网站语音功能：需走 **https://8.130.190.169**（自签名证书，浏览器会提示安全警告，需先手动接受一次）
>
> 以下测试步骤中的 `localhost:3000` / `localhost:8000` 均为本地开发地址；对生产环境测试时替换为上述生产地址即可。

## 三、详细测试步骤

### T1 — 博主个人网站首页

**目标**：验证网站能正确展示博主信息、项目、视频等。

**操作步骤**：

1. 打开浏览器，访问 http://localhost:3000（本地开发模式）、http://localhost:8000（本地生产模式）或 http://8.130.190.169:8000（远程生产服务器）
2. 观察页面是否加载出博主头像、姓名「钟笑咪」、个人简介
3. 滚动页面，检查以下区块是否存在且有内容：
   - 「关于我」/ 个人简介区
   - 项目列表（应看到「妙喵 Creator Agent」「AI Tutorial Demo」等项目卡片）
   - 视频列表（应看到至少 2 个视频条目）
   - 日记预览区（应看到最近几条日记标题和日期）
   - FAQ / 常见问题区
   - 联系方式（邮箱、GitHub 链接）
4. 点击导航栏中的各链接（项目、日记、博客等），确认页面跳转正常

**预期结果**：
- 所有区块正确渲染，无空白或报错
- 项目卡片有标题、阶段、标签
- 点击视频能跳转到对应平台
- 页面整体风格一致，无明显排版错乱

### T2 — 妙喵聊天助手对话

**目标**：验证聊天机器人能正常响应，包括 FAQ 匹配、日记查询、LLM 改写。

**操作步骤**：

1. 打开网站首页，在**右下角**找到猫咪浮动按钮（🐾图标）
2. 点击按钮，弹出聊天窗口，妙喵应显示欢迎语：「你好，我是妙喵...」
3. **测试 FAQ 问答** — 依次输入以下问题，观察回答：

| 输入内容 | 预期回答方向 |
|----------|-------------|
| `你是谁` | 介绍自己是博主的数字分身 |
| `博主是谁` / `谁是钟笑咪` | 介绍钟笑咪的身份 |
| `有什么项目` | 列举已完成的项目 |
| `怎么联系` | 提供邮箱等联系方式 |
| `最近在做什么` | 提及最近的日记内容 |

4. **测试日记查询** — 输入：
   - `最近在忙什么`
   - `今天做了什么`
   - `日记`
   
   预期：妙喵应提及最近的日记条目（如 6 月底的日记）

5. **测试 LLM 风格改写** — 输入一个开放性问题：
   - `你对 AI 有什么看法`
   
   观察回答是否带有博主的个人风格（而非纯模板回复）

6. **测试视频上下文** — 如果页面上有视频，在聊天时带上视频上下文：
   - 输入 `这个视频讲了什么`
   
   观察是否引用了视频字幕内容

7. **测试离线回退** — 如果未配置 LLM Key：
   - 断开网络或使用无效 Key
   - 输入问题，观察是否回退到本地 FAQ 匹配
   - 回答应以「猫猫暂时连不上后端大脑」开头

**预期结果**：
- 每次对话都有回复（不超时）
- FAQ 类问题匹配准确
- 日记查询能返回最近日记内容
- LLM 回复自然，不是生硬的模板
- 无 500 错误或空白回复

### T6 — Chrome 插件视频问答

**目标**：验证 Chrome 插件能在 B 站视频页注入猫咪私教，并支持语音输入。

**前置操作（首次）**：
1. 打开 Chrome，访问 `chrome://extensions/`
2. 打开右上角「开发者模式」开关
3. 点击「加载已解压的扩展程序」
4. 选择 `E:\Create\Code\ewa\extension` 目录
5. 确认「妙喵私教」扩展出现在列表中

**测试步骤**：

1. 打开 B 站视频：`https://www.bilibili.com/video/BV1mJ4m147PG`
2. 等待页面加载完成，视频右下角会出现妙喵猫咪气泡
3. 点击气泡展开面板，检查是否显示：
   - 视频标题
   - 状态徽章「练习模式」
   - 课程总关卡数（5 关）
4. **注册视频**：插件会自动向后端注册当前视频，检查面板是否显示视频标题和字幕数量
5. **视频问答**：
   - 在视频播放到某个时间点时（如 0:30），在插件输入框中提问
   - 输入 `什么是正当防卫`
   - 观察回答是否包含视频字幕上下文
   - 回答末尾是否有 `[SEEK:秒数]` 跳转提示
6. **语音输入**：
   - 按住输入框右侧的 🎤 按钮
   - 说出 `什么是正当防卫`
   - 松开后应显示「识别中…」，然后自动发送识别文本
   - 观察妙喵是否能基于语音识别结果作答
7. **点击跳转**：如果回答中有跳转建议，点击后视频应跳转到对应时间点
8. **LLM 不可用时**：
   - 如果未配置 API Key，回答应来自字幕搜索或本地 FAQ
   - 回答应标注「offline: true」

**预期结果**：
- 插件面板正确注入到 B 站页面
- 能识别当前视频并加载字幕/课程
- 文字问答有视频上下文相关性
- 语音输入能识别中文并自动发送
- 跳转功能正常
- 不影响 B 站页面本身的正常使用

### T7 — 课程闯关答题

**目标**：验证课程加载、答题评分、进度持久化的完整流程。

**前置**：需要先完成 T6（确保视频已注册）。

**测试步骤**：

1. **加载课程** — 在插件面板中，如果当前视频有对应课程（BV1mJ4m147PG 对应「正当防卫的构成要件」课程），面板应显示课程信息：
   - 课程标题：正当防卫的构成要件 — 罗翔刑法
   - 总步骤数：5
   - 当前步骤：第 1 步「起因条件：必须存在不法侵害」

2. **第 1 步 — 正确回答**：
   - 阅读题目：关于张三误伤物业工人的案例
   - 输入正确答案（包含「不构成正当防卫」「假想防卫」「不存在不法侵害」等关键词）
   - 提交后观察：
     - 是否显示「通过」
     - 是否获得星星（首次通过 = 3 星）
     - 妙喵反馈是否指出关键要点
     - 是否自动跳转到第 2 步

3. **第 1 步 — 语音答题**：
   - 重新进入第 1 步或从第 2 步回到第 1 步
   - 按住 🎤 按钮说出答案：「不构成正当防卫，因为这是假想防卫，没有现实的不法侵害」
   - 松开后自动识别并提交
   - 观察是否能正确评分并给出反馈

4. **第 2 步 — 错误回答再重试**：
   - 故意输入一个错误答案（如只写「正当防卫」）
   - 观察：
     - 是否显示「未通过」
     - 妙喵是否指出漏答的要点
     - 是否显示「⏪ 回退到视频片段」建议
     - 星星是否减少（重试通过 = 2 星或 1 星）

5. **继续完成所有 5 步**：
   - 逐步完成剩余步骤
   - 观察进度条/星星/鱼/成长值的变化

6. **刷新页面后恢复**：
   - 完成部分步骤后，刷新浏览器页面
   - 重新打开插件面板
   - 确认之前的进度（星星数、完成步骤、当前步骤）已恢复

**预期结果**：
- 课程正确加载，步骤信息完整
- 文字答题评分准确（关键词匹配 + LLM 语义判断）
- 语音答题能被识别并正确提交
- 通过/未通过的反馈信息清晰
- 星星、鱼、成长值正确累加
- 进度持久化到 SQLite，刷新后不丢失

### T10 — 视频模糊查询（网站妙喵）

**目标**：验证网站妙喵能按模糊描述检索视频，并基于真实字幕回答、给出精准时间戳跳转。

**操作步骤**：

1. 打开网站（本地 http://localhost:8000 或生产 http://8.130.190.169:8000 ），点击右下角猫咪按钮打开妙喵聊天窗口
2. **模糊查询** — 输入：`罗翔的正当防卫视频你有什么看法？`
3. 观察回答：
   - 返回类型应为 `video_query`，即识别出这是在问某个视频
   - 回答内容应基于该视频字幕原文，不编造字幕之外的案例（如不应再出现"昆山龙哥案"这类幻觉）
   - 回答中应带 `[mm:ss]` 时间戳，并出现「跳到 mm:ss」按钮
4. 点击「跳到 mm:ss」按钮，确认跳转落点对应回答中提到的时间点（而非固定跳到视频开头）
5. **当前视频总结** — 访问 http://8.130.190.169:8000/projects?video=BV1mJ4m147PG 页面，在妙喵中输入：`这个视频讲了什么`
6. 观察回答：
   - 应总结当前页面视频（BV1mJ4m147PG 正当防卫）的内容
   - 总结要点应来自字幕，且附时间戳

**预期结果**：
- 模糊查询能命中正确视频并给出基于字幕的回答
- 无字幕外编造的案例
- seek 按钮落点与回答中的时间戳一致
- 在带 video 参数的页面能正确识别"当前视频"

## 六、常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 页面空白 | 前端未启动或端口不对 | 确认 `npm run dev` 在运行，访问 3000 端口 |
| API 返回 404 | 后端未启动 | 确认 `python run.py` 在运行 |
| 聊天无回复 | LLM Key 未配置 | 在 `.env` 中设置有效的 API Key |
| 聊天返回 500 | LLM API 调用失败 | 检查 Key 是否有效，网络是否通畅 |
| 插件无反应 | 后端未启动或 URL 配置错误 | 确认后端可访问 `http://8.130.190.169:8000/health`；若用本机后端，修改 `extension/content/*.js` 中的 `API_BASE` 并重新加载插件 |
| 语音输入无反应 | 麦克风权限被拒绝 | 在 B站页面地址栏左侧允许麦克风权限；或检查 Chrome 站点设置 |
| 课程加载失败 | 课程 JSON 文件缺失 | 检查 `server/data/miaomiao/lessons/` 目录是否存在 `lesson_luoxiang_001.json` |
| 管理后台 401 | Token 不正确 | 查看后端启动日志中的随机 Token |
| 数据库锁定 | 多进程同时写入 | 关闭其他使用同一 DB 的进程 |
